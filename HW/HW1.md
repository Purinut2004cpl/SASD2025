ในคลิป Video Martin Fowler พูดถึงหัวข้อ Not Just code Monkeys ว่าสิ่งกำลังเกิดขึ้นในวงการ Agile คือ คนเขียนโปรแกรมมักถูกมองเป็นแค่คนพิมพ์โค้ดตามคำสั่ง (code monkeys)
ทุกวันนี้หลายองค์กรทำงานแบบนี้คือฝั่งธุรกิจคิด → เขียน requirement → ส่งให้นักพัฒนา → นักพัฒนาก็แค่ทำตาม
เหมือนเป็น “เครื่องพิมพ์โค้ด” หรือที่เขาเรียกว่า Code Monkeyแต่ Martin Fowler บอกว่า มันไม่ใช่แนวคิดของ Agile ที่แท้จริงAgile จริง ๆ ควรเป็นการ “คุยกัน”
ไม่ใช่โยนเอกสารข้ามกำแพงมาให้ dev ทำอย่างเดียว นักพัฒนาควรมีสิทธิ์ถามว่า   ทำไปเพื่ออะไร, ผู้ใช้เจอปัญหาอะไรจริงๆ , วิธีนี้แก้ปัญหาได้ดีที่สุดไหม
เพราะหลายครั้ง business อาจอธิบายปัญหาผิดจุด

แต่คนเขียนโปรแกรมกลับเห็นวิธีแก้ที่ดีกว่าได้อีกเรื่องที่เขาพูดแรงพอสมควรคือความรับผิดชอบของคนทำซอฟต์แวร์เราไม่ควรพูดว่า“เขาสั่งมา ผมก็แค่ทำตาม”
เช่น ถ้าระบบถูกออกแบบมาให้หลอกผู้ใช้กดผิด สมัครโดยไม่รู้ตัว หรือยกเลิกยากมากถึงจะเป็นคำสั่งจากบริษัท นักพัฒนาก็ยังมีส่วนรับผิดชอบอยู่ดีเขาอยากให้ dev มองตัวเองเป็น
“มืออาชีพ” ไม่ใช่แค่ลูกจ้างเขียนโค้ดเพราะซอฟต์แวร์ที่เราสร้างมันส่งผลกับชีวิตคนจริง ๆสุดท้าย Martin Fowler อยากให้ทุกคนจำไว้ว่านักพัฒนาที่ดี ไม่ได้เก่งแค่ภาษาโปรแกรมแต่ต้องเข้าใจคน เข้าใจปัญหา และกล้าคิด กล้าถามเราไม่ใช่ code monkeyเราเป็นคนที่ช่วย “แก้ปัญหา” ด้วยเทคโนโลยี
